# Debugging History

## Issue 1 — Incorrect `set_start_state()` Usage

### Error

```text
TypeError:
set_start_state(): incompatible function arguments
```

### Cause

`set_start_state()` requires a named argument (`robot_state=`). More importantly, the current robot state should not be obtained directly from `RobotModel`.

### Solution

Replace

```python
arm.set_start_state(robot_state=current_state)
```

with

```python
arm.set_start_state_to_current_state()
```

---

## Issue 2 — Incorrect `set_goal_state()` Usage

### Error

```text
TypeError:
set_goal_state(): incompatible function arguments
```

### Cause

The installed MoveItPy version does not support passing a dictionary of joint positions directly.

### Solution

Create a `RobotState` object and assign the joint positions.

```python
goal_state = RobotState(robot_model)

goal_state.set_joint_group_positions(
    "fr3_arm",
    np.array(goal_positions)
)

arm.set_goal_state(
    robot_state=goal_state
)
```

## Issue 3 — Reading Current Robot State

### Error

```text
AttributeError:
RobotModel has no attribute
get_current_state()
```

### Cause

`RobotModel` only stores the robot model.

It does **not** provide the live robot state.

### Solution

Use

```python
arm.set_start_state_to_current_state()
```

instead.

## Issue 6 — Planning Succeeds but Execution Fails

### Error

```text
Goal request rejected
```

Planning completed successfully, but the controller rejected the trajectory.

---

### Investigation

Verify the controller.

```bash
ros2 control list_controllers
```

Verify the action server.

```bash
ros2 action info \
/fr3_arm_controller/follow_joint_trajectory
```

Verify hardware interfaces.

```bash
ros2 control list_hardware_interfaces
```

Verify joint names.

```bash
ros2 param get \
/fr3_arm_controller joints
```

---

## Issue 7 — Isolate MoveIt from the Controller

To determine whether the problem originated from MoveIt or the controller, send a trajectory directly to the controller.

```bash
ros2 action send_goal \
/fr3_arm_controller/follow_joint_trajectory \
control_msgs/action/FollowJointTrajectory ...
```

If this succeeds, the controller is functioning correctly and the problem lies in the trajectory generated by MoveIt.

## Issue 8 — Missing Time Parameterization

### Symptoms

```text
Goal request rejected
```

### Cause

The generated trajectory did not contain proper timing information.

### Solution

Configure the OMPL response adapters.

```python
response_adapters = (
    "default_planning_response_adapters/"
    "AddTimeOptimalParameterization "
    "default_planning_response_adapters/"
    "ValidateSolution "
    "default_planning_response_adapters/"
    "DisplayMotionPath"
)
```

---

## Issue 9 — Missing Planning Parameters

### Warning

```text
Parameter
plan_request_params.max_velocity_scaling_factor
not found
```

### Solution

```python
plan_request_params = {
    "planning_pipeline": "ompl",
    "planner_id": "RRTConnectkConfigDefault",
    "planning_time": 5.0,
    "max_velocity_scaling_factor": 0.05,
    "max_acceleration_scaling_factor": 0.05,
}
```

---

## Issue 10 — Planning Plugin Not Found

### Error

```text
Planning plugin name is empty
```

### Cause

The OMPL configuration was overwritten.

### Solution

Merge the YAML configuration before adding custom parameters.

```python
ompl = load_yaml(...)

ompl.update({
    "planning_plugins": [
        "ompl_interface/OMPLPlanner",
    ],
})
```

# Key Lessons Learned

- Use `set_start_state_to_current_state()` instead of reading the current state from `RobotModel`.
- Use a `RobotState` object when calling `set_goal_state()`.
- Verify the planning group before debugging planning failures.
- Verify controllers and hardware interfaces before assuming a MoveIt problem.
- Test the `FollowJointTrajectory` action directly to isolate controller issues.
- Enable **AddTimeOptimalParameterization** to generate executable trajectories.
- Configure velocity and acceleration scaling parameters explicitly.
- Ensure the OMPL configuration is merged correctly without overwriting existing settings.
